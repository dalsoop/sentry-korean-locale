# sentry-korean-locale

Community-maintained **Korean locale** (`ko.js`) for Sentry self-hosted.

- **15,173 entries** — from 199 official → expanded ~76×
- **1.3 MB** compiled `ko.js` chunk (drop-in replacement)
- Translation engine: Google TranslateGemma 27B (BF16, llama.cpp, 4× RTX 3090 tensor-split)
- Compatible with **Sentry 25.10.x** (chunk hash `ko.711dae3eefae37a1.js`)

Generated by [gemma-translate](https://github.com/dalsoop/gemma-translate) + `phs-sentry-i18n` pipeline.

## Quick Apply (docker container swap)

```bash
git clone https://github.com/dalsoop/sentry-korean-locale
cd sentry-korean-locale

# local docker host
sudo bash scripts/deploy.sh

# Proxmox LXC-wrapped docker
sudo bash scripts/deploy.sh --pct 50105
```

## Cache Busting (optional)

Browsers aggressively cache chunks by content-hash in filename. If a user already visited the old `ko.711dae…`:

```bash
sudo bash scripts/bust-cache.sh          # generates new hash + rewrites app.js
```

## What's Inside

```
dist/
  ko.js              # 1.3 MB — drop-in chunk
  ko.js.gz           # 610 KB

src/
  translations.json  # 1.9 MB — clean {msgid: msgstr} dictionary
  state.json         # 2.9 MB — full pipeline state with origin tags
  glossary.example.json  # 105 common UI terms seed (Save/Cancel/etc.)

scripts/
  deploy.sh          # docker cp + nginx reload
  bust-cache.sh      # rename + app.js rewrite for guaranteed cache invalidation
```

## Translation Origin Tags (`state.json`)

- `existing` — Sentry official ko (1,408 entries)
- `translated` — TranslateGemma 27B (13,511)
- `glossary` — standardized via glossary table (36)
- `manual` — hand-edited (2)
- `skip-svg` — SVG path data, no translation (195)
- `skip-tz` — moment.js timezone data (21)

## Translation Quality Notes

- **UI labels / buttons**: excellent (Save → 저장, Cancel → 취소)
- **Mid-length sentences**: generally natural
- **Long explanations**: occasionally the model summarizes or adds a bit; still readable
- **Placeholders** (`%s`, `{name}`, `[tag]`): mostly preserved; rare reordering on very short strings

To hand-correct a term, edit `src/translations.json` and rebuild (see [gemma-translate](https://github.com/dalsoop/gemma-translate) for the build pipeline) — or simply swap into `dist/ko.js` and rerun deploy.

## Regenerating from Scratch

```bash
# Prerequisite: gemma-translate + a running llama-server instance
# (see https://github.com/dalsoop/gemma-translate)

phs-sentry-i18n --sources "ja,ru,ro,cs,fr,de,es" extract
phs-sentry-i18n scan             # also extract from compiled JS
phs-sentry-i18n translate --workers 16
phs-sentry-i18n glossary-apply   # standardize via glossary
phs-sentry-i18n build
phs-sentry-i18n --bust deploy
```
